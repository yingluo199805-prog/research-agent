# 年报附注科目分析（按比亚迪模型模板）

把上市公司年报附注中的五个科目做成带勾稽校验、比率反推和预测假设的分析表，
复用比亚迪模型（`收纳\01_公司个股\比亚迪\深度研究\【广发汽车】比亚迪模型-260527.xlsx`）
中已验证的表格结构。

## 核心理念

这套表的价值不在抄数，而在**勾稽**：每张表都让模型算出来的数和年报披露的数对账，
"应=0"的校验行暴露口径差异（减值、重分类、企业合并）。预测期则把历史反推出的
隐含比率（折旧率/转固率/摊销率/资本化率）锁定为可调假设，让附注数据驱动利润表预测。
所以：**校验不平时标注原因，绝不硬调数字凑平**；所有预测假设都要留在深红色可调单元格里。

**但记住一条血的教训：内部校验行全部=0，不等于数取对了。** 如果某行从一开始就取错了来源
（如把"利润表研发费用"当成"费用化研发投入"），它和同源的其它行照样内部自洽、校验行照样=0，
但整列对不上年报。所以自检不能只看"应=0"行，必须**另外抽样把建模值与年报截图上的披露数直接对一遍**
（见第5步）。这是这套技能最容易自欺的地方。

## 五张表与适用范围

| Sheet名 | 年报附注 | 预测期 | 规格文件 |
|---|---|---|---|
| 固定资产 | 固定资产附注 | 有（默认3年） | `references/fixed-assets.md` |
| 在建工程 | 在建工程附注 | 有 | `references/cip.md` |
| 无形资产 | 无形资产附注 | 有 | `references/intangibles.md` |
| 资本化研发 | 研发投入情况表(MD&A)+开发支出/研发费用附注 | 有 | `references/capitalized-rd.md` |
| 政府补助分析 | 其他收益/递延收益/非经常性损益附注 | **无（只做历史）** | `references/gov-subsidy.md` |

支持三种场景：
- **A 新公司建表**：从年报PDF起步，五张表全做或用户指定子集
- **B 年度/中报更新**：已有表追加一年实际值列，预测列右移、比率重锁
- **C 单表模式**：只做某一张（如只做政府补助），输出独立文件

## 工作流程

### 第0步：确认参数

开工前**必问时间跨度**——这是唯一每次都要主动确认的参数，因为它直接决定要下载几份年报、
建几列，事后返工成本高。问法举例："做近几年？默认按过去 8 年年报来。"
用户明确给了范围就用用户的；用户说"随便/你定/没特别要求"就用**默认近 8 年**。

**"近8年"的终点 = 最新一个已披露的年报年度，按当前日期推算，不要想当然锚到"去年"。**
A股/港股年度报告通常在次年 1-4 月披露完毕。判断规则：设当前年份为 Y、月份为 M——
- 若 M ≥ 5（5月及以后）：年度 Y-1 的年报必已发布，终点 = **Y-1**，区间 = Y-8 … Y-1；
- 若 M ≤ 4（年初）：Y-1 年报可能尚未出齐，终点取 **Y-2**（保守），先建表，待 Y-1 年报披露后用场景B补一年。

举例：当前 2026-06 → 终点 2025 → 默认区间 **2018-2025**（含刚披露的 2025 年报）。
拿不准某年年报到底出没出，就去下载源翻一眼最新一期年报的年度，以实际能下到的为准。

其余参数用户没说就用默认值、不要逐条追问，开工后在交付摘要里说明所用默认即可：
- 做哪几张表：默认按用户话里点到的科目；用户说"全套/附注分析"则五张全做；
- 输出位置：默认写入用户指定的公司模型 xlsx（新增 sheet）；没有模型文件则**所有科目（含政府补助）
  做进同一个工作簿** `{公司}_附注科目分析_{起}-{终}.xlsx`，每科目一个 sheet——
  **政府补助不再单独出文件**，和其余四张并在一起（用户偏好，2026-06 起固定）。
  仅当用户明确要求"单独出某张表"时才拆分单文件；
- **默认输出文件夹**：`C:\Users\Lenovo\Desktop\收纳\01_公司个股\{公司}\深度研究\`
  （与比亚迪模型同级；公司文件夹不存在就新建 `{公司}\深度研究\`）。
  截图底稿 `{公司}_年报附注底稿.xlsx` 与下载的年报PDF放同目录下的 `研究底稿\` 子文件夹。
  用户另指定路径时以用户为准。
- 预测年数：**默认 3 年**（政府补助除外），且预测值要按逻辑先填好（见第3步），不留空给用户。

### 第1步：下载年报PDF

用 `web-access` 技能联网。优先公司官网投资者关系页；找不到再用巨潮资讯网
（A股，cninfo.com.cn）或港交所披露易（hkexnews.hk）。只需要**年度报告**
（更新场景加中报）。下载到公司文件夹的研究底稿目录下，文件名保留年份。
注意：H股公司同时有中文版年报，优先下中文版；附注科目数据在"财务报表附注"章节。

### 第2步：定位附注页 → 截屏底稿

这一步是留痕环节，**不可跳过**。底稿的用途是"方便人工核对"：审表人不必翻几百页 PDF，
在一个 Excel 里逐年看到**原始附注页的整页截图**，旁边标注年报/附注/页码。
所以底稿存的是**页面截图**，不是文本摘录。

**底稿做成一个 Excel**（整次任务只有一个底稿文件，不是每科目一个）：
`{公司}_年报附注底稿.xlsx`，每个科目一个 sheet，sheet 内截图**按年份升序**排列。

操作（定位思路：**先吃透一年，再套到其他年**）：

同一家公司的年报模板逐年高度稳定——附注编号、章节顺序、各表叫法基本不变。所以不要每年从零盲扫，
而是先在一个"锚定年"把所有科目找准、记下"附注地图"，再把它套用到其余年份：

1. **锚定年（取最新一年，与当前最相关）彻底定位一次**：
   打开最新年报，先读「财务报表附注」开头的**编号清单**（A股"(六/七) 合并财务报表项目注释"会把
   1货币资金…14固定资产…逐条列出，一两页就能拿到所有科目的附注编号）。据此跳到每个科目附注页确认，
   再找研发投入情况表（管理层讨论·研发投入小节）和政府补助各口径（其他收益/递延收益/营业外/非经常性）。
   记下"附注地图"：{科目 → 附注编号 + 该年页码 + 所在章节}。各科目在年报里的位置、A股 vs 港股结构差异
   见 `references/note-locator.md`——**建表前务必读它**。
2. **把附注地图套到其余年份**：同名附注在各年都在、编号至多偏移一两位、页码随版面平移。
   用锚定年的科目名/编号在每年搜该附注标题即可快速命中，逐年只渲染/读那一两页确认。
3. **定位脚本是助手，不是关卡**：`python scripts/locate_notes.py <PDF或文件夹>` 可对每年每科目
   返回候选页（页顶"编号+科目名"标题为主信号），适合做第2步的批量套用。
   **但若某科目/某年脚本命中不准或为空（如港股嵌入式附注、研发投入情况表港股没有），
   不要在定位上死磕——直接回退到读该年「附注编号清单」那一两页，找到后继续。** 定位只是手段，目的是拿到正确页。
4. 页码一律用 **PDF 物理页码**（从1开始），不是版心印刷页码——两者常差几页，定位后翻看截图核对。
   不靠固定编号取数（附注编号每年会变），靠科目名认表；表格跨页时把相关页都纳入。
   注意：资本化研发的"研发投入情况表"在管理层讨论章节、不在财务附注，单独定位、也要截进底稿。
5. 把定位结果写成 spec.json，调用本技能脚本生成截图底稿：
   ```
   python scripts/build_digao.py spec.json
   ```
   spec.json 结构、字段含义见 `scripts/build_digao.py` 文件头注释。脚本纯本地渲染、不联网，
   按年份升序把每页渲染成图嵌入对应 sheet，标题行自动写"{年份}年报 — {附注} — {PDF名} — 第X页"。
6. 截图底稿同时**是建表的取数依据**：第3步往表里填的每个历史硬编码值，都应能在底稿对应年份的
   截图里指认到。建表时若发现某数在底稿里找不到出处，说明页码定位漏了，回本步补截。

### 第3步：结构化建表

读对应科目的 `references/*.md` 规格文件，按规格在目标工作簿中建sheet。
通用规范（颜色RGB、数字格式、布局）见 `references/conventions.md`，建表前先读它。

关键适配原则：
- **类别行数自适应**：披露几类资产/几个项目就生成几行，勾稽公式范围跟着行数走。
  比亚迪固定资产是5类、在建工程按7个工业园；别家可能是4类、按具体工程项目。
  在建工程项目数超过10个时，取期末余额前9大、其余合并为"其他项目"。
- **跨表联动只在这5张表内部建**：在建工程转固预测 → 固定资产"CIP转入"；
  资本化研发"转入无形资产" → 无形资产"内部研发资本化"。
  比亚迪模型里挂到其他sheet的引用（如收入分配拆分驱动研发投入），
  在新公司一律降级为深红色独立假设值，旁边批注说明默认取值逻辑，避免断链。
- **历史值全部硬编码**（深蓝），来源列写"附注{编号}"；BS披露值行用于校验。
- **单位跟年报披露口径**（千元或元），在第2行注明；政府补助汇总区可用亿元。
- **预测值要先按逻辑填好、不留空**：预测期假设单元格（深红）默认按各科目规格里的逻辑
  自动取值（折旧率/摊销率锁近2年平均、购置增长率默认0%或按扩张趋势、转固率取近3年平均等），
  预测列的金额由公式引用这些假设算出。交付的是一张**已经填满、能直接看结果**的表；
  深红只是标明"这些格子用户可以改"，不是"留给用户填"。每个非零默认假设在旁边列写一句取值理由。

### 第4步：重算

重算让公式产生缓存值，否则后面没法读值自检。

**重算方法（按本机可用项选）**：`xlsx` 技能的 `scripts/recalc.py` 依赖 LibreOffice，
本机（Windows，未装 LibreOffice）跑不了——不要在它身上耗时间。改用以下任一：
- **Excel COM**（本机装了 Excel，首选）：`win32com.client` 打开→`Application.CalculateFull()`→保存；
- **formulas 包**（无 Excel 时）：`formulas.ExcelModel().loads().calculate()` 回写。

> ⚠️ **铁律：重算并保存必须是对该文件的最后一次写操作。**
> openpyxl 保存时会丢弃所有公式的缓存值。如果你"先重算、再用 openpyxl 打补丁/改格式"，
> 缓存值会被全部抹掉，交付的文件在 Excel 打开前所有公式格显示空白、自检也读不到值。
> 正确顺序：**先把所有 openpyxl 编辑（建表、改格式、打补丁、第5步自检的修正）全部做完 →
> 最后用 Excel COM 重算并保存**。若自检后又用 openpyxl 改了东西，必须再跑一次 COM 重算收尾。

### 第5步：完工自检 → 改错

建完表**一定要自己过一遍再交付**，不要把没核对的表丢给用户。读取重算后的缓存值，逐项检查并就地修正：

1. **公式错误清零**：全表扫描有无 `#REF!/#DIV/0!/#VALUE!/#NAME?/#N/A`。
   常见根因：把中文说明误写成 `=…` 公式、跨表引用单元格错位、范围没跟着类别行数走——逐个改掉。
2. **建模值 vs 年报披露 抽样比对（最关键，必做）**：从每张表各挑几个关键历史硬编码值
   （如某年期末原价/累计折旧/合计、费用化与资本化投入、递延收益期末），回到该年的**截图底稿**
   逐个核对数字一致。**这是"应=0"校验行抓不到的一层**——取错来源的行内部照样自洽。
   重点盯易混口径：费用化研发投入≠利润表研发费用、计提折旧≠利润表折旧费用、各政府补助口径。
   对得上才算这张表过关；对不上回底稿改源。
3. **校验行逐年看**："应=0"行的实际值：
   - =0 → 通过；
   - 小额差异且能归因（减值准备、外币折算、重分类）→ 在校验行旁批注原因，保留；
   - 无法归因 → 回截图底稿核对原始数字（这正是底稿存在的意义），改对；仍不平则**保留差异并在交付时点名**。
4. **比率体检**：抽查隐含比率是否落在合理区间（折旧率3%-25%、转固率30%-90%、摊销率2%-30%、
   资本化率0-50%），越界的先怀疑抄错数或单位错位（A股"元"vs 港式"千元"差三个数量级），回底稿查。
   反推比率若能和年报直接披露的比率对上（如资本化率 vs 披露的"资本化比重"），顺手比一比，最稳。
5. **预测列体检**：预测年份金额是否都已算出（没有空白、没有继承历史空值）；
   假设单元格是否都是深红、是否都有取值理由；预测出的期末值与末年实际相比有没有突兀的断崖/暴涨。
6. **联动体检**：固定资产"CIP转入"是否确实 `=在建工程!…`、无形资产"内部研发资本化"是否
   `=资本化研发!…`；改了在建工程/资本化研发的假设，固定资产/无形资产是否跟着变。

改完若动了公式或格式（用 openpyxl），**务必重新执行第4步重算收尾**（见那条铁律），再复查一遍受影响的格子——
否则自检的修改会把缓存值抹掉。交付前最后确认：随便挑几个公式格，确认有缓存值（不是空白）。
**自检发现并修掉的问题，在交付摘要里如实说一句。**

### 第6步：交付摘要

向用户报告：每张表的勾稽结果（几年全平/哪年差多少、原因）、
反推出的关键比率及其趋势、披露中的特殊事项（重分类、企业合并、口径变化）、
预测假设默认值清单（提醒用户深红色单元格可调）。

## 年度更新场景（场景B）要点

1. 下载新年报，定位新一年附注页，把新年份的截图按升序补进既有底稿 Excel
   （在 spec.json 里只列新增年份、重跑 `build_digao.py` 生成补充底稿，或并入既有底稿）；
2. 在最后一个实际值列后插入新列：新实际值硬编码（深蓝）、原预测首年列删除或右移；
3. 隐含比率反推公式扩展到新年份，预测假设的"锁定近2年平均"窗口随之滚动；
4. 全表重算 + 完整自检（第5步），校验行必须覆盖新年份；
5. 对比新实际值与原预测值的偏差，在交付摘要中给出预测复盘（偏差>10%的科目要点名）。

## 常见坑

- A股年报单位多为"元"、港式披露多为"千元"，混用会导致校验差三个数量级；
- 固定资产"本期计提折旧"≠利润表折旧费用（有制造费用资本化进存货的部分），不要互相校验；
- 政府补助的"计入当期损益"有四个去向（其他收益/营业外收入/冲减财务费用/冲减成本），
  漏一个口径就对不上非经常性损益附注；
- 在建工程"其他减少"科目常含转入投资性房地产/无形资产，不全是转固；
- 2021年起部分公司执行新租赁准则，使用权资产可能从固定资产中拆出，注意口径连续性。
