# 通用规范（建表前必读）

从比亚迪模型实测提取的格式标准，五张表统一执行。

## 布局

- 内容从 **B列** 开始（A列留白做边距），第1-3行表头说明区，第5行年份轴；
- 列结构：B=科目名，C=附注/来源，D起=历史年份（如 `2018A`...`2025A`），
  随后预测年份（`2026E`...`2028E`），假设区右侧一列（如O列）放假设理由文字；
- 政府补助表无预测列，年份轴只到最新实际年；
- 区块编号用中文数字章节（一、二、三…），章节内子假设用 8.1/8.2 式编号；
- 每张表末尾固定有"备注与说明"区：勾稽关系、特殊事项、预测假设说明。

## 表头三行

```
B1: 附注 {编号} {科目} — {一句话概括}（{起}-{终}）        ← 14pt 加粗 深蓝字
B2: 单位：{千元} ｜ 颜色：深蓝=年报实际 ｜ 深红=可调假设 ｜ 黑=公式 ｜ 黄底=合计 ｜ 绿底=校验
B3: 勾稽：{本表核心勾稽等式，一行}
```

## 颜色与字体（精确RGB）

| 用途 | 规格 |
|---|---|
| 全表字体 | Microsoft YaHei（微软雅黑）10pt |
| 大标题（B1） | 字体 `FF1F4E78` 深蓝、加粗、14pt |
| 年份表头行（第5行） | 白字 `FFFFFFFF` 加粗，填充深蓝 `FF1F4E78` |
| 区块标题行（一、二、…） | 加粗，填充浅蓝 `FFD9E2F3` |
| 年报实际值（硬编码） | 字体深蓝 `FF1F4E78` |
| 可调预测假设（硬编码） | 字体深红 `FFC00000` |
| 公式 | 字体黑 `FF000000` |
| 合计行 | 加粗，填充浅黄 `FFFFE699` |
| 校验行（应=0） | 填充浅绿 `FFE2EFDA` |

## 数字格式

| 内容 | numfmt |
|---|---|
| 金额（千元/元） | `#,##0;\(#,##0\);\-` （负数加括号、零显示为-） |
| 比率/占比 | `0.0%` 或 `0.00%`（折旧率等小比率用两位小数） |
| 年份表头 | 文本，如 `2024A`、`2026E` |

## 公式书写原则

- 合计 = `SUM()` 范围公式，范围必须随类别行数自适应；
- 校验行 = 模型计算值 − 披露值，永远不要写成 0 的硬编码；
- 比率反推统一包 `IFERROR(...,"")`，避免早年分母为0时报错；
- 滚动表期初 = 上年期末的单元格引用（首年期初硬编码自年报）；
- 预测列公式引用假设区单元格，**不在公式里写死参数**；
- 跨表引用只允许指向本技能生成的5张表之间（如 `=在建工程!L170`），
  其余一律降级为深红独立假设；
- **空格一律用 0 或真正空白(None)，绝不写空字符串 `''`**：滚动表/反推里常有 `=-D68`、`=D16+D64-D16`
  这类公式引用基准年(首列)单元格，若那格被写成 `''`（空字符串），Excel 对空串做算术会报 `#VALUE!`，
  并沿整张滚动表级联成几十上百个错误（实测一个空串 D68 级联出 162 个 #VALUE!）。
  openpyxl 写表时，留空的单元格要么填 0、要么不赋值（None），不要赋 `''`。
- **"来源"列(C列)是文字标签，不能以 `=` 开头**：写成 `=-固定资产!CIP转入合计` 会被 Excel 当公式解析、报 `#NAME?`。
  来源说明就用纯文本，如"桥接自固定资产8.4"。

## 底稿与模型的关系

底稿存原始（按年按页的**附注页截图**+页码出处，由 `scripts/build_digao.py` 生成，
整次任务一个 Excel、每科目一个 sheet、年份升序），模型表存结构化结果。
模型表中每个区块的 C 列"附注/来源"写明出处（如"附注16(2)"），
让任何数字两步可溯源：模型 → 附注编号 → 底稿对应年份截图 → 年报原文。

## 重算（Windows 本机）

`xlsx` 技能的 `scripts/recalc.py` 依赖 LibreOffice，本机未装、跑不通，别用它。
改用 Excel COM（`win32com.client` → `CalculateFull()` → 保存）或 `formulas` 包重算。
重算只是为了让公式产生缓存值，方便随后读值自检。
**顺序铁律：重算并保存必须是对该文件的最后一次写操作**——openpyxl 保存会丢弃公式缓存值，
所以所有 openpyxl 编辑（建表/改格式/自检修正）全部做完后，最后才用 COM 重算保存；
自检若又改了东西，必须再跑一次 COM 收尾。详见 SKILL.md 第4步。
