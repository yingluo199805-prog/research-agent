# 附注定位指引（省token的关键）

在 300+ 页的年报里逐页找附注是最大的 token 浪费。核心策略：**先吃透一个锚定年、再套到其他年**，
定位脚本只是辅助、搞不定就回退读年报——不要在定位上死磕、更不要通读整本 PDF。
本文件讲这套策略、各科目在年报里的位置、A股/港股结构差异、以及定位脚本怎么用。

## 〇、定位总策略：锚定年优先

同一家公司的年报模板逐年高度稳定（附注编号、章节顺序、各表叫法基本不变），所以：
1. **取最新一年做锚定年**，先读「财务报表附注」开头的**编号清单**（A股"(六/七) 合并财务报表项目注释"
   一两页内把所有科目附注号列全），据此跳到各科目确认页码，加上研发投入情况表(MD&A)与政府补助各口径，
   记下"附注地图"：{科目 → 附注编号 + 页码 + 章节}。
2. **把地图套到其余年份**：同名附注各年都在、编号至多偏移一两位、页码随版面平移；
   按科目名/编号搜该年附注标题即可命中，逐年只读那一两页确认。
3. **定位脚本是助手不是关卡**：命中不准或为空时，直接回退到读该年「附注编号清单」那一两页找到为止。
   定位只是手段，目的是拿到正确页——别为自动化而自动化。

## 一、定位脚本（辅助批量套用）

```
python scripts/locate_notes.py <存年报PDF的文件夹>
```
对每份 PDF、每个科目，返回候选页（`best` 最可能、`top_pages` 备选、`with_span` 含跨页）。
脚本原理三层叠加：① 读 PDF 书签目录；② 科目名广义同义词（简繁多种叫法）命中；
③ **页顶「编号+科目名」标题**为强信号（A股"14、固定资产"、港股"16. 物業、廠房及設備"都是编号附注，
通用），辅以通用滚动表特征词（期初/期末/本期增加/累计折旧/账面原值/减值准备…，跨公司不变）密度排序。

**脚本是"缩范围"工具，不是"定答案"工具**：拿到候选页后，渲染/读取该页确认确实是目标附注表
（看页顶标题、看是不是带分类别明细的滚动表），再喂给 `build_digao.py`。命中为空或不对时按下面的
结构知识回退、或回到锚定年策略读编号清单。

## 二、各科目在年报里的位置（A股标准结构）

A股年报第十节「财务报告」→「财务报表附注」→「(六/七) 合并财务报表项目注释」是编号清单
（1货币资金…14固定资产…），每个科目一节、按编号排，**这是五张表里固定资产/在建工程/无形资产/
开发支出/政府补助各口径的取数处**。研发投入情况表是例外（见下）。

| 科目 | 在年报哪一节 | 定位锚点 / 注意 |
|---|---|---|
| 固定资产 | 财务附注·项目注释 | "NN、固定资产" 起始页 → 分类别(房屋/机器/运输/电子…)原值·累计折旧·减值滚动表，常跨2-3页 |
| 在建工程 | 财务附注·项目注释 | 紧邻固定资产之后；含按项目/工业园滚动、工程物资、预算追踪 |
| 无形资产 | 财务附注·项目注释 | 分类别(土地使用权/专利/软件/客户关系…)原值·累计摊销滚动 |
| 开发支出 | 财务附注·项目注释 | 资本化研发的"中转站"科目；按项目滚动(期初/本期增加/转无形/转损益/期末) |
| 研发费用(按性质) | 财务附注·费用类注释 | 职工薪酬/物料/折旧摊销/检测费…拆分；区别于利润表"研发费用"一行 |
| **研发投入情况表** | **管理层讨论/董事会报告·研发投入小节**（**不在财务附注**） | 证监会规定格式，含"本期费用化研发投入/本期资本化研发投入/研发投入合计/资本化比重"。**这是资本化研发表 section二 的唯一权威来源**（见 capitalized-rd.md 铁律） |
| 政府补助·其他收益 | 财务附注·损益类注释 | "其他收益"含政府补助与增值税加计抵减（后者非补助，要分列） |
| 政府补助·递延收益 | 财务附注·负债类注释 | "递延收益"滚动表（期初/新增/转出/期末），与资产相关补助主要在此 |
| 政府补助·营业外收入 | 财务附注·损益类注释 | "营业外收入"中与政府补助相关部分 |
| 政府补助·非经常性损益 | 财报前部·非经常性损益明细表 | 用于经常性/非经常性拆分互验 |

## 三、港股（HKFRS）差异 —— 脚本对这些会返回 None，需手动回退

港股年报无证监会编号格式，结构和用词都不同，**不要套A股的位置**：
- **在建工程**：通常**没有独立附注**，是「物業、廠房及設備（PP&E）」附注里的一列（在建/在建工程/
  Construction in progress）。脚本对"在建工程"可能命中会计政策噪音页 → 回退到 PP&E 附注页（best 命中的"固定资产"页）里找在建列。
- **无形资产**：可能并入 PP&E 后或单列「無形資產」，注意是否含商誉。
- **开发支出/资本化研发**：港股叫「開發成本」，可能在无形资产或单列；**无"研发投入情况表"**，
  资本化信息散见于无形资产附注与会计政策，需读原文判断，section二的"披露比重"口径在港股不一定有。
- **政府补助**：多并入「其他收入及收益」「遞延收入」，无"营业外收入/非经常性损益"这两个A股口径。
- 港股常用**千元**且为繁体，注意单位与简繁（建表单位跟年报口径，见 conventions.md）。

## 四、定位 → 截图 → 建表 的衔接

1. `locate_notes.py 文件夹` → 得到每年每科目候选页；
2. 逐年渲染/读候选页**确认**（页顶标题对不对、是不是带分类别明细的那张表），跨页表把相邻页一并纳入；
3. 把确认后的 {年份, PDF, 科目, 页码} 写进 `build_digao.py` 的 spec.json，一次生成截图底稿；
4. 同一批确认页就是建表的取数依据——建表时每个历史硬编码值都应能在该页截图里指认到。
