# 资本化研发 sheet 规格

原型：比亚迪模型「资本化研发」sheet（附注八 研发支出 + 开发支出）。
本表是链路表：研发投入 → 费用化（研发费用）/ 资本化（开发支出）→ 转入无形资产。

## 表头

```
B1: 附注 {编号} 研发支出（按性质 + 费用化/资本化 + 项目滚动）
B2: 链路：研发投入 → 费用化 → 研发费用 / 资本化 → 开发支出 → 转入无形资产 ｜ 单位：{千元}
```

## 区块结构

**一、研发投入按性质拆分**
披露几项做几行（常见：职工薪酬/物料消耗/折旧及摊销/检测费/股份支付/其他）+ 合计行。
历史硬编码。预测列：比亚迪模型挂在「收入分配拆分」sheet——**新公司降级为深红假设**
（默认按研发费用率×预测收入，或近2年增速外推，O列写取数逻辑）。
公司不按性质披露的，本节降级为"研发投入合计"单行。

**二、费用化 vs 资本化**
行：费用化研发支出｜资本化研发支出｜合计核对（=两者之和）｜资本化率（`=IFERROR(资本化/合计,"")`，预测列深红可调）。
预测：资本化支出 `=合计×资本化率`，费用化 `=合计-资本化`。

> **数据来源铁律——必须取年报"研发投入情况表"，不是利润表研发费用。**
> A股年报"董事会报告/管理层讨论与分析"里有一张固定的"研发投入情况表"，
> 直接披露四个数：**本期费用化研发投入 / 本期资本化研发投入 / 研发投入合计 / 研发投入资本化的比重(%)**。
> 本节费用化(row 费用化)、资本化(row 资本化) **逐字取这张表的"投入"口径**，于是：
> - 合计核对行 `=费用化+资本化` 必然 = 披露的"研发投入合计"（这是硬校验，差应=0）；
> - 资本化率 `=资本化/合计` 必然 = 披露的"资本化比重"（可逐年比对验证）。
>
> **绝不能用"利润表研发费用"行充当"费用化研发支出"。** 利润表研发费用 = 费用化研发投入
> + 已资本化开发支出的当期摊销 + 其他，通常**显著大于**费用化研发投入（随资本化存量增大差距越大），
> 用它会让合计虚高、资本化率被系统性低估、研发投入/营收虚高——这正是最易踩的坑。
> 若分析上确实想保留利润表研发费用，单独加一行 memo，不要塞进费用化/资本化的拆分里。

**三、开发支出（资产负债表科目，按项目）**
披露的项目维度（比亚迪：汽车/电池/手机项目）各一行 + 期末合计。仅历史硬编码。

**三-2、开发支出滚动表（核心项目，历史+预测）**
行：期初余额 / 本年新增（内部开发资本化）/ 转入无形资产(-) / 计入当期损益(资本化失败转出)(-) /
期末余额 / 校验行（期初+新增-转入-计入损益-期末，应=0）/ 历史转入率（`=本年转入/上年期末`）。
预测假设（深红）：本年新增默认按最新年量级或×增长系数；
转入无形资产默认 `=(期初+0.5×新增)×转入率`（转入率取历史平均）；计入损益默认0。
**"转入无形资产"行是无形资产sheet 9.3的引用源。**

**四、研发占收入比**
`=研发投入合计/营业总收入`（收入可硬编码在公式里并注明出处，或引用模型收入行——
后者只在用户模型里有标准收入行时使用）。

**五、备注**
按发现写编号条目：资本化率历史走势及拐点解读（这是判断利润质量的关键信号）、
披露口径变化年份、待补数据、与其他表的校验关系、会计政策（摊销方法/年限）。
最后加【两步流程】【预测逻辑】【联动】三段固定说明。

## 易错点

- "研发投入"（按性质合计）与"费用化+资本化"两套披露可能对不上（口径含/不含股份支付），
  对不上时保留差异并备注，常见原因是部分年份口径变化；
- 开发支出余额≠当年资本化金额：资本化先进开发支出池，达到可使用状态才转无形资产，
  两步不要混；
- 早年年报可能不披露资本化拆分（比亚迪2020-2021缺失），缺失年份留空并在备注"待补"，
  不要插值编数。
