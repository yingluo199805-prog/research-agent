# 在建工程 sheet 规格

原型：比亚迪模型「在建工程」sheet（附注16，按7个工业园滚动，2018-2025+预测）。
设 P = 披露的项目维度数。比亚迪按工业园披露；多数公司按具体工程项目披露。
P>10 时取期末余额前9大项目逐项做，其余合并"其他项目"块。

## 表头

B1/B2 同通用规范；B3 不强制（本表勾稽较多，写在备注）。

## 区块结构

**一、CIP总额勾稽（与资产负债表对账）**
行：在建工程（账面价值）｜工程物资｜合计｜BS披露在建工程NBV｜校验行（应=0）。
注意BS"在建工程"科目是否含工程物资因公司而异，先对账确定口径再定结构。
预测列：在建工程 `=七.期末`；工程物资默认持平上年。

**二、CIP期末账面价值 — 按项目（P行）**
历史硬编码 + 合计 + 校验行（vs 一.账面价值，应=0）。仅历史。

**三、重要在建工程变动滚动表（项目 × 年度）**
每个项目一个块，固定7行：
```
期初余额 / 本年增加 / 重分类(如有) / 企业合并(如有) / 本年转入固定资产(-) / 其他减少 / 期末(公式=前6行之和)
+ 校验行: 期末公式值 - 二.披露期末（应≈0，差异为减值，标注金额）
+ 转固率: =IFERROR(-转入/(期初+本年增加+重分类+合并),"")
```
重分类/企业合并行只在披露中实际出现的年份填值，没有就保留为0行（结构统一便于跨项目比较）。
最后【全部项目 合计】块：期初/本年增加/转入/期末各合计行 + 整体转固率。

**四、预算追踪 — 工程投入占预算比例（披露值）**
每项目两行：预算数、投入占预算比例（年报附注直接披露，硬编码）。
公司不披露预算的，整节省略并在备注说明。

**五、CIP→PP&E 转固分配 — 按资产类别**
来源=固定资产sheet的CIP转入历史值（房屋/机器/运输/办公各一行）+ 转固合计 +
校验行（vs 三.转固合计，应=0）+ 各类占比行（`=类别/合计`）。
这节是把"按项目"的CIP和"按资产类别"的PP&E接起来的桥，是联动的关键。

**六、预测假设输入（深红可调）**
- 6.1 各项目本年新增CIP投入预测（默认=近2-3年平均×衰减系数，O列写资本开支判断理由）+合计
- 6.2 整体转固率假设（默认近3年平均）
- 6.3 转固分配比例假设——按资产类别（默认 `=AVERAGE(近3年占比)`）

**七、CIP滚动预测 + 转固预测**
- 7.1 CIP滚动表（合计）：期初/本年增加(=6.1合计)/转固(`=-(期初+增加)×6.2`)/期末
- 7.2 转固按资产类别拆分预测：每类 `=-转固×6.3占比`，**这些行是固定资产sheet 8.4的引用源**；
  土地类转入默认0（土地走购置不走CIP），单独一行注明。

**八、说明与备注**
【数据范围】【勾稽关系】【转固率历史走势解读】【资产类别分配趋势】【预测逻辑】【可调节假设清单】【局限】。
局限里写清楚：项目级预测是参数化估计；减值差异金额；土地转入按0处理。

## 易错点

- 滚动表"期末"用公式算，和披露期末的差额就是当年减值计提——这是校验行存在的意义，不要把披露值直接抄进期末行；
- "其他减少"可能含转投资性房地产/无形资产，金额大时在备注点名；
- 项目维度跨年可能改名/合并（比亚迪"东北工业园"2023年并入"其他"），用重分类行处理并备注。
