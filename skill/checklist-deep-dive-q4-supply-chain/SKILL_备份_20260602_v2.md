## 通用规范（开工前通读）

### 总原则

1. **客观性第一**。对每个问题同时列出支持证据与反驳证据；对公司"不良记录"与"护城河"要以同等严肃的标准检视。
2. **量化优先**。能用数字的地方不要用形容词——"市占率高" → "2024 年 CR1 56%，CR5 93%"；"ROE 不错" → "过去 5 年 ROE 均值 28%、剔除杠杆 ROIC 22%"。
3. **最终产出**：最终报告必须是一份独立可读的完整文档。每个子问题（如 4.1、4.2）在报告中应保留完整的论证过程、关键数据表格和情景分析。**篇幅控制按输出格式区分**：
   - **Markdown 格式**（.md 初稿或 chat 纯文本输出）：全文总字符数 **30,000-42,000**（含标题、表格、空行、引用标记），其中纯中文字数不低于 **15,000 字**。低于下限说明内容压缩过度需补充；超出上限优先压缩重复论证，而非删关键数据表。
   - **Word/PDF 终版**：总页数严格控制在 **20-30 页**（A4，11pt 字体）。
   - **HTML 可视化报告**：不以字符数 / 页数约束，改按**信息密度**约束——每个子问题（4.1–4.7）至少含 1 张数据表或 1 个可视化组件 + 完整论证；所有关键数字必须落到表格或图里、不靠纯文字堆砌。低于此密度说明可视化偷懒。
4. **公司特定专题（按需增设）**：若公司当前存在一个**主导性的产业链相关事件或议题**（如反垄断/政策冲击、技术拐点、重大供给侧变化、关键诉讼），应在最相关的子问题下**增设一个独立专题深挖**（含时间线、多源量化、可比案例对标、整改/演化预期）。这类专题是 checklist 的合理延伸，不要因为它不在 4.1–4.8 固定问法里就略过——产业链研究的价值恰在于抓住当下最关键的结构性变量。

判断是否存在可访问的公司本地文件夹（用户提供路径，或工作目录下有"3-参考研究"子目录）：有则进入**本地模式**，无则进入**Chat 纯联网模式**。下面"报告结构与命名"、"输出流程"、"研究方法"三段均按此模式分支。

### 报告结构与命名

- 报告命名："{公司名}-CheckList深挖-Q{N}-{专题名}"
- 存放位置：
  - **本地模式**：公司文件夹下 "3-参考研究\4-AI研究成果\Claude-CheckList-深挖"，子文件夹不存在时自动创建
  - **Chat 纯联网模式**：不落盘，直接在对话中输出；若用户指定路径则按指定路径保存
- 报告结构：**摘要 → 各子问题逐一展开 → 核心待解决问题（3-5 个）→ 研究结论（500 字以内）→ 来源清单附录**
  - **摘要**：用 **3-4 张"核心判断卡片"**（每张一个一句话标题 + 一段量化论证）+ **一张核心 KPI 速览表**呈现，而非纯段落。判断卡片是摘要的标准形态。
  - **来源清单附录**（标准件）：报告尾部按信息层级分组列出关键来源——一级（一手数据）/ 二级（公司口径）/ 三级（卖方与专家）/ 联网补盲，每组列出具体文件名或篇名。正文仍保持内联引用，附录是汇总索引。

### 输出流程

**本地模式（三阶段）**

1. **初稿 .md**：完成研究后输出为 Markdown 文档，存入 "Claude-CheckList-深挖" 文件夹
2. **用户反馈 + 自检迭代**：用户审阅后可能提出修改意见或要求 Claude 自检（最多 4 轮）。自检重点：事实准确性、数据引用出处、论证逻辑自洽、关键反面证据、**总字符数是否在 30,000-42,000 区间**
3. **终版转格式**：用户确认 .md 终版后，用 docx skill 生成 Word 文档，再转为 PDF。格式美观简洁，确保中文排版正常

**Chat 纯联网模式（单阶段）**

直接在对话中输出完整 Markdown 正文，无文件落盘、无 docx skill 调用。用户反馈后在对话内迭代，不产出 .docx/PDF 终版。

**HTML 可视化模式（与上述数据模式正交，由用户指定或当目标是"给人看的网页报告"时采用）**

终版交付为一份独立可读的 HTML 报告，而非 Word/PDF。要点：
1. **样式规范**：调用 `finance-web-style` skill，采用冷灰蓝学术/金融风（白底、衬线标题、等宽数字字体、左侧 sticky 目录 TOC）。
2. **必备可视化组件**（按子问题对号入座，不要纯文字堆砌）：
   - 摘要：Hero KPI bar + 3-4 张判断卡片 + 核心 KPI 速览表
   - 4.1 全景结构：分层 tier-flow 图（L1→Ln 逐层）
   - 4.2 加价拆解：每个样本产品一条**堆叠条形图** + 成本/加价明细表
   - 4.3 链主分润：关键公司**毛利率/ROE 横向条形对标图**
   - 4.4 财务特征：各环节 KPI 卡片组 + **"附加值 vs 资本回报"象限 SVG 图**
   - 4.5 议价权/专题：隐性条款表 + （若有公司特定事件）**事件 timeline**
   - 4.7 推演：**三情形卡片（悲观/基础/乐观 + 概率）** + ±20% 敏感性条形 + 历史冲击参照框
3. **落盘**：本地模式存入 "Claude-CheckList-深挖" 文件夹；Chat 模式按用户指定路径保存。HTML 模式下不再额外产出 docx/PDF。
4. **自检**：HTML 模式自检按"信息密度"而非字符数（见总原则第 3 条 HTML 分支）。

### 研究方法：信息源（按模式分支）

**本地模式**：双路协同
- **本地文件（基本盘）**：公司文件夹内的公告、财报、研报、专家访谈、竞对资料、行业数据等。主体材料，优先使用——每个子问题先基于本地材料，消化后再补充其他来源
- **联网搜索（补盲 + 交叉验证）**：以下情况必须联网——本地缺最近 1-2 个季度数据、需竞对公开信息而本地无竞对文件夹、需第三方行业数据（App Annie / SimilarWeb / 协会 / 统计局）、需验证本地卖方观点时效性。联网信息额外标注"联网获取"

**Chat 纯联网模式**：联网搜索承担基本盘角色。仍遵守信息分级；无法获取的信息（如未公开专家访谈、细分研报）显式标注"本轮研究未覆盖"，不编造

**信息交叉验证原则**（两种模式共用）：当信息指向不同结论时，优先信任一手数据（公司公告、财报原文）> 本地高质量研报/专家访谈 > 联网搜索结果 > 卖方观点

### 信息分级与引用格式

- **信息分级**（与项目通用 PI、公司专用 PI 保持一致）：统一采用 **一级（一手数据）/ 二级（公司口径）/ 三级（卖方与专家观点）/ 推断（研究员推断）** 四档编号，每条关键论点必须标明属于哪一档；联网获取的信息额外加 **"联网"** 标签（与四档正交，可叠加，如"三级/联网"）。
- **引用规范**：每个关键数据点需注明信息来源类型和信息层级，让读者能判断可信度。格式示例：'FY2025收入624亿（4Q25业绩会，一级）'、'佣金率约15-20%（多篇专家纪要综合，三级）'、'罚款预估20-40亿（瑞银研报，二级）'。不需要精确到页码，但不接受'根据财报'或'根据公开信息'这类无法区分来源的模糊表述。对于推断性结论，必须标注'推断'并简要说明推理依据。巴芒材料仅用于内化判断框架，报告正文中不需要引用巴芒原文或标注巴芒材料出处。引用规范仅适用于公司研究材料（财报、业绩会、研报、专家纪要等）。

### B/M 透镜

巴菲特把"链主 / 收费桥（toll bridge）"作为生意质量的核心之一——他在可口可乐、美国运通、蓝筹邮票、喜诗糖果身上都在找"独占性的收费环节"。1993 年股东信指出这些公司的"品牌、产品特质与配销通路"构成了 **"高耸的护城河来保卫其经济城堡"**（来源：`股东信-1993.md`）。芒格则强调 **"invert, always invert"** 的产业链思维——看一个行业不要只顺着商流想，也要**从最终消费者付出的每一块钱反推**每一环各拿到多少、为什么能拿到这么多，以及这个分配关系的**脆弱点**在哪里；同时用 1994 USC 演讲中的 **"competitive destruction happens again and again"**（来源：`Poor Charlie's Almanack` line 3437）提醒研究者，产业链格局不是静态的，**历史上每一个看似稳固的链主都可能因技术、需求、渠道的结构性变化被颠覆**。

判断关键：① 在整条产业链的利润分配里，公司拿到的份额是否与其无可替代性匹配；② 这家公司对所在产业链的上下游是否能持续强势？是否这种强势程度很难受时间、技术变革、外部环境影响而减弱？

### 4.1 需求锚定 + 产业链全景透视

- **(a) 最底层的需求**：这个行业最终服务的对象是谁？他们最本质的需求是什么？这个需求**如何定义了整条产业链的形状**——为什么是当前这个结构而不是其他结构？
- **(b) 全景结构化描绘**：从最上游到最下游依次列出参与者、商流路径、每一环的核心职能（研发 / 原材料 / 加工 / 制造 / 集成 / 渠道 / 服务 / 售后）。目标是让**一个完全不了解这个行业的人**通过这一段也能看清整个价值链的结构、参与者、利益分配与商业逻辑。
- **(c) 各环节的不可或缺性**：对每一环用一句话概括"它的生意本质 + 为什么是上下游无法替代的"。

### 4.2 价值与成本流动：终端产品的加价与损耗拆解（Munger 反向推演）

以行业里一个典型终端产品 / 服务为例，**从最上游开始**逐环拆清楚：

- 每一环的成本是多少？
- 每一环加价了多少金额、加价率是多少？
- 中间有没有损耗 / 漏损（报废率、退货、库存减值）？

让一个人**一眼看明白终端售价是如何构成的**。然后反推：哪一环拿走了终端价格的最大份额？这个份额与该环节的附加值、壁垒是否匹配？哪一环的利润份额**看起来拿多了**、哪一环**看起来拿少了**——为什么没有被压缩 / 没有被抬升？

### 4.3 链主识别与权力结构

判断产业链中哪一方是**绝对意义上的"链主"**，拥有最多话语权：

- **(a) 强势的根源**：该环节为什么强势？是技术独占、牌照、规模、品牌、位置、关系资源、还是客户心智？**其他环节为何无法提升议价能力**——壁垒在哪？
- **(b) 链主对成本结构与利润分配的改造**：链主的存在如何挤压了上下游的利润？哪些环节被迫让利、哪些环节被迫承担风险？用具体数字说明（如链主毛利率 vs 上下游平均毛利率的差距）。
- **(c) 权力结构变化的触发条件**：在什么情境下链主可能被颠覆——技术迭代、渠道更替、消费偏好变化、政策变化、全球供应链风险、上游资源稀缺性变化？历史上（本行业或类似行业）是否出现过链主更替？

### 4.4 各环节的竞争壁垒与财务特征（逐环节分析）

按"上游 → 中游 → 下游"的顺序，对每个**核心环节**严格按以下 3 个维度分析：

- **(a) 环节的核心职能**：这一环节存在的必要性是什么？具体做什么（研发、制造、加工、集成、渠道、服务）？解决了上游什么问题？为下游提供了什么价值？
- **(b) 竞争与格局**：在这个环节脱颖而出需要什么核心优势？**行业壁垒是什么**（技术、资本、认证、规模、品牌、专利、牌照、关键人、关键客户）？真正体现技术含量或专业能力的是什么？**集中度如何**？集中度高 / 低的原因？头部几家企业有怎样的特点？
- **(c) 商业模式与财务特征**：这个环节的典型企业，**利润表 / 资产负债表 / 现金流量表**普遍呈现什么特征？用具体维度：毛利率、净利率、存货周转、应收周转、资产负债率、固定资产 / 总资产占比、Capex 强度、OCF / 净利润、FCF 质量。**为什么**会呈现这些特征？与行业特性或商业模式有何关联？

### 4.5 议价权地位 + 真实世界的谈判与利益交换

按 4.4 的环节顺序，依次分析每个环节**如何"谈生意"**——除了价格之外，真实商业世界里常见的**隐性条件、附加条款、博弈方式与利益交换**：

- **(a) 隐性条款清单**：比如年降要求、返利、账期拉长、票据结算（银承 / 商承）、二供三供体系、排他协议、最低采购量（MOQ）、联合研发、模具费承担、渠道返利、返点、库存责任、售后责任划分、保密协议、独占协议、战略合作承诺、"未来订单承诺"或画饼绑定——这个环节里哪些条款是常态？
- **(b) 强弱关系如何影响谈判**：谁更容易让步？让到哪里？为什么？讨价还价的**关键砝码**是什么（规模、技术、产能、替代性、认证周期、交付速度、品牌、客户资源）？
- **(c) 惯例 vs 强压**：哪些条款是行业惯例（基本不可逆）？哪些是强势方强行压出来的（一旦力量对比反转就会消失）？
- **(d) 风险转嫁**：库存风险、坏账风险、召回风险、研发投入、价格波动——**哪些风险被转嫁给谁**？被转嫁方有没有对冲手段？
- **(e) 利益互换的典型套路**：拿订单换降价、拿未来承诺换专供、拿账期换现金折扣、拿产能优先权换技术协同、拿独占协议换联合研发投入……

### 4.6 过去 10 年的演变与结构性重排（用事实说话）

必须使用**可靠的数据、指标变化、重大行业事件**支撑结论，不要抽象描述：

- **(a)** 技术进步、成本结构变化、效率提升带来了哪些结构性重排？（具体事件 + 关键数据）
- **(b)** 各关键环节的**集中度、议价权、利润率、竞争门槛**如何变化？
- **(c)** 产业链是否出现**利润池迁移、角色重心移动、链主更替**？
- **(d)** 上下游力量博弈是否发生显著变化（账期、供需平衡、替代性、定价权）？

基于以上"发生过的事实"，总结：**过去 10 年这个行业为何演化为当前格局**？

### 4.7 未来 5-10 年推演 + 对这家公司的具体影响

- **(a) 持续的结构性力量**：哪些力量会持续影响产业链——技术、政策、需求、全球供应链、资本结构、海外竞争？
- **(b) 权力重分配预期**：哪些环节可能**增强或削弱议价权**？是否可能诞生**新链主**？上下游是否会重新整合？利润池是否有再次迁移的可能？
- **(c) 关键风险事件**：哪些潜在但关键的**黑天鹅 / 灰犀牛**可能导致格局剧烈变化？
- **(d) 产业链稳定结构判断**：用紧凑逻辑呈现 **"过去事实 + 未来趋势"** 的完整推演，形成对未来 5-10 年产业链"稳定结构"的判断。
- **(e) 三情形推演（标准动作）**：给出 **悲观 / 基础 / 乐观** 三种情形，每种**标注主观概率**（合计 100%），并写明该情形下的关键假设（反垄断/政策、技术/AI、竞争、海外业务等）与对公司核心指标（收入增速、take rate、利润率中枢）的结论。
- **(f) 回到这家公司 + 定量敏感性**：在未来稳定结构中，公司处于什么位置？是赢家、输家、还是被动参与者？对最主要的 2-3 个外生变量（原材料价格、汇率、政策、技术、佣金率、上下游结构）做 **±20% 敏感性测算**，给出对公司收入 / 毛利率 / 净利润的**定量影响**（HTML 模式下用敏感性条形图呈现）。
- **(g) 历史冲击参照**：过去是否出现过类似量级的冲击（疫情、政策大改、技术替代等）？当时对业绩的实际影响多大？**恢复用了多久**？据此给出本轮冲击的恢复时长判断。
